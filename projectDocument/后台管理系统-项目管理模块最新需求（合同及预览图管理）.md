# 后台管理系统-项目管理模块最新需求（合同及预览图管理）

# 一、需求概述

本次最新需求针对后台管理系统「项目管理模块」，新增合同及预览图管理相关功能，核心涵盖基础字段新增、联动区域展示、文件上传、云端存储及接口适配，所有功能需与现有系统逻辑、布局、样式保持统一，数据库表及字段由Gemini自行设计，确保与现有系统无缝对接，不破坏原有功能。

# 二、具体最新功能需求

## 2.1 基础项目信息字段新增

- 新增字段：在「基础项目信息」区域中，新增「是否有合同」「是否有预览图」两个字段，展示位置固定在原「订单金额」字段后方，与现有字段布局协调、样式统一。
- 字段属性：两个字段均为单选类型，默认值均为「否」，交互方式采用现有系统单选弹窗样式（点击字段弹出单选选项，选择后关闭弹窗，展示选中结果），与现有系统单选交互逻辑完全一致。
- 编辑权限：未结清项目可正常切换选择「是」/「否」；已结清项目不可编辑，字段设为只读状态，与基础项目信息其他字段只读样式一致。

## 2.2 项目合同管理区域（联动「是否有合同」字段）

- 区域显示逻辑：当「是否有合同」选择「是」时，在「成本支出管理」区域下方显示「项目合同管理」区域；选择「否」时，隐藏该区域，无任何展示痕迹。
- 区域样式：布局结构、边框、内边距、标题样式与「成本支出管理」「资金管理」区域完全一致，标题固定为「项目合同管理」，与现有模块命名风格统一。
- 文件上传规则：
  - 支持文件类型：图片（所有主流图片格式，与凭证上传图片格式一致）、PDF文件，两种类型可单独上传，也可同时上传。
  - 上传限制：PDF文件仅支持单个上传，最大支持10M；图片支持多图上传，无数量限制（贴合凭证上传逻辑），单张图片大小限制与凭证上传一致。
  - 上传交互：支持文件拖拽上传、点击选择上传，上传过程中显示进度条，上传成功后展示文件预览（图片可直接预览，PDF显示封面预览），支持删除已上传文件。
- 字段切换删除逻辑：若用户先选择「是」并上传合同文件，后续切换为「否」，需自动删除云端存储的该项目所有合同文件及对应文件夹，确保云端无残留文件，删除操作需记录日志（包含删除文件路径、删除时间、操作人）。
- 创建校验规则：当「是否有合同」选择「是」时，创建单子（新建/补录）需校验是否已上传合同文件（至少上传1个文件，图片或PDF均可），未上传则弹出校验提示（提示内容：“请上传合同文件后再创建项目”），禁止创建，校验风格与现有系统一致。

## 2.3 预览图管理区域（联动「是否有预览图」字段）

- 区域显示逻辑：当「是否有预览图」选择「是」时，在页面最下方显示「预览图管理」区域；选择「否」时，隐藏该区域，无任何展示痕迹。
- 区域样式：布局结构、边框、内边距、标题样式与其他区域完全一致，标题固定为「预览图管理」，与现有模块命名风格统一。
- 文件上传规则：
  - 支持文件类型：仅支持图片（所有主流图片格式，与凭证上传图片格式一致），不支持其他文件类型，上传时自动校验格式，非法格式弹出提示。
  - 上传限制：最多上传4张图片，支持多图同时上传，单张图片大小限制与凭证上传一致。
  - 上传交互：与合同文件上传交互一致（拖拽/点击上传、进度条、预览、删除），上传满4张后，隐藏上传入口，提示“最多可上传4张预览图”。
- 字段切换删除逻辑：若用户先选择「是」并上传预览图，后续切换为「否」，需自动删除云端存储的该项目所有预览图及对应文件夹，确保云端无残留文件，删除操作需记录日志（包含删除文件路径、删除时间、操作人）。
- 创建校验规则：当「是否有预览图」选择「是」时，创建单子（新建/补录）需校验是否已上传预览图（至少上传1张），未上传则弹出校验提示（提示内容：“请上传预览图后再创建项目”），禁止创建，校验风格与现有系统一致。

## 2.4 区域布局顺序

- 若「是否有合同」「是否有预览图」均选择「是」，则两个区域按以下顺序排列：成本支出管理区域 → 项目合同管理区域 → 预览图管理区域（预览图管理区域在合同管理区域下方），区域之间间距与现有区域间距一致，排版流畅无错位。
- 若仅其中一个选择「是」，则对应区域按上述对应位置单独展示，无多余空白。

## 2.5 云端存储路径规范

- 项目合同文件存储路径：固定格式为 `/project-contract/${年月日}/${项目名称}/contract_随机数.文件格式`，示例：/project-contract/20260411/中国八建展会/contract\_67877.png、/project-contract/20260411/中国八建展会/contract\_12345.pdf；其中「年月日」为文件上传当天日期（格式：YYYYMMDD），「项目名称」为当前项目的名称（与项目信息中一致），「随机数」为6-8位随机数字，「文件格式」为上传文件的原始格式（小写）。
- 预览图存储路径：固定格式为 `/project-previw-images/${年月日}/${项目名称}/previw_随机数.图片格式`，示例：/project-previw-images/20260411/中国八建展会/previw\_67877.png；参数规则与合同文件一致，仅路径前缀、文件名前缀不同，图片格式保留原始格式（小写）。
- 路径注意事项：项目名称中若包含特殊字符，需进行转义处理，确保路径合法，避免云端存储异常；文件夹自动创建（上传文件时，若对应年月日、项目名称文件夹不存在，自动创建），删除文件时，若文件夹为空，同步删除对应空文件夹。

## 2.6 上传实现逻辑

- 合同文件、预览图的上传实现逻辑，严格参考现有系统「单据凭证图片上传」的逻辑，包括但不限于：文件校验（格式、大小）、上传进度展示、上传失败重试、上传成功回调、文件预览、文件删除等逻辑，确保上传功能稳定，与现有系统上传体验一致。
- 补充说明：图片支持多图上传（与凭证图片多图上传逻辑一致），PDF文件仅支持单个上传，上传时需单独校验PDF文件大小（不超过10M），超出则弹出提示“PDF文件最大支持10M，请压缩后重新上传”。

## 2.7 接口适配调整（新增/调整，贴合现有接口风格）

- 项目创建/编辑接口：
  - 新增参数：isHasContract（是否有合同，“是”/“否”，默认“否”）、isHasPreview（是否有预览图，“是”/“否”，默认“否”），与前端字段对应，禁止传入非法值。
  - 新增校验逻辑：创建项目时，若isHasContract=“是”，需校验合同文件上传状态（至少1个文件）；若isHasPreview=“是”，需校验预览图上传状态（至少1张），校验失败返回对应异常提示。
  - 新增同步逻辑：编辑项目时，若isHasContract从“是”改为“否”，触发合同文件及对应文件夹删除逻辑；若isHasPreview从“是”改为“否”，触发预览图及对应文件夹删除逻辑，删除成功后同步更新接口返回状态。
- 合同文件上传/删除接口（新增）：
  - 上传接口：路径、请求/响应格式参考凭证上传接口，新增参数file（文件）、projectId（项目ID），返回文件存储路径、上传状态，上传过程中按规范生成存储路径，自动创建对应文件夹。
  - 删除接口：路径、请求/响应格式参考凭证删除接口，传入参数filePath（文件存储路径）、projectId（项目ID），删除云端文件，若对应文件夹为空，同步删除文件夹，返回删除结果。
- 预览图上传/删除接口（新增）：
  - 上传接口：参考凭证上传接口，新增参数file（图片文件）、projectId（项目ID），校验图片格式、数量（不超过4张），按规范生成存储路径，返回上传状态、文件路径。
  - 删除接口：参考凭证删除接口，传入参数filePath（图片存储路径）、projectId（项目ID），删除云端图片，若对应文件夹为空，同步删除文件夹，返回删除结果。
- 项目详情查询接口：
  - 新增返回参数：isHasContract、isHasPreview（字段值“是”/“否”），合同文件列表（包含文件路径、文件名称、文件类型、上传时间），预览图列表（包含图片路径、上传时间），用于前端区域展示及文件预览。

# 三、交互与样式要求（完全贴合现有系统，无需新增风格）

- 只读字段样式：新增的「是否有合同」「是否有预览图」字段，已结清项目时设为只读，样式与现有系统只读字段完全一致（如灰色显示、无编辑边框、hover无编辑提示），不新增额外样式。
- 合同及预览图管理区域样式：两个新增区域的标题样式、边框、内边距、间距与现有区域完全一致；文件上传入口、进度条、预览样式、删除按钮样式，均参考现有凭证上传样式，确保视觉统一。
- 单选弹窗交互：「是否有合同」「是否有预览图」的单选弹窗，样式、弹窗大小、选项排列、选中效果与现有系统单选弹窗完全一致，点击弹窗外部可关闭弹窗，无额外交互。
- 校验提示：所有异常校验提示（文件大小、格式、数量，创建校验等），样式（颜色、边框、位置）、提示语气与现有系统完全一致，不突兀，确保用户体验统一。
- 响应式适配：所有新增区域、字段，需适配现有系统响应式布局，在电脑端、平板端等不同屏幕尺寸下，布局流畅、无错位、无样式错乱。

# 四、开发注意事项（聚焦最新需求落地，无数据库设计相关内容）

1. 现有系统兼容性：新增的功能、接口，需与现有系统无缝对接，不破坏现有任何功能（如项目创建、成本支出管理、凭证上传等），确保现有系统正常运行，不出现线上故障。
2. 接口兼容性：新增/调整的接口，需兼容现有前端调用逻辑，不改变现有接口的请求/响应格式，仅新增必要参数、返回值，确保前端无需额外修改即可适配（除合同及预览图区域展示相关前端调整外）。
3. 交互一致性：所有新增交互（单选弹窗、文件上传、校验提示等），需与现有系统交互风格完全一致，不新增额外交互方式，确保用户操作无学习成本。
4. 文件上传及存储：严格遵循存储路径规范，确保文件路径合法，特殊字符需转义；上传逻辑参考凭证上传，确保上传稳定，支持重试机制；切换字段删除文件时，需确保云端文件及对应空文件夹完全删除，无残留，删除操作记录日志。
5. 文件校验：严格执行上传文件的格式、大小、数量限制，PDF文件单独校验大小（不超过10M），预览图校验数量（不超过4张），非法文件及时弹出提示，禁止上传；创建项目时，严格校验合同、预览图的上传状态，未达标禁止创建。
6. 区域联动：确保「是否有合同」「是否有预览图」字段切换时，对应管理区域的显示/隐藏实时生效，无延迟；两个区域同时存在时，布局顺序正确，无错位、重叠。
7. 测试覆盖：开发完成后，需覆盖最新需求所有场景测试，包括：字段切换、文件上传/删除、云端存储、接口调用、创建校验、异常场景模拟，确保功能稳定可用。

# 五、核心开发要点

- 新增字段：基础项目信息新增「是否有合同」「是否有预览图」，默认否，单选弹窗交互，已结清项目不可编辑。
- 合同管理：选是显示区域，支持图片+PDF上传（PDF单文件10M），选否删除云端文件，创建需校验上传。
- 预览图管理：选是显示区域，最多4张图片上传，选否删除云端文件，创建需校验上传；与合同区域并存时在下方。
- 存储路径：严格遵循合同、预览图的云端存储路径规范，上传逻辑参考凭证上传。
- 接口适配：新增合同/预览图上传/删除接口，调整创建/编辑/详情接口，贴合现有接口风格，新增校验及同步逻辑。
