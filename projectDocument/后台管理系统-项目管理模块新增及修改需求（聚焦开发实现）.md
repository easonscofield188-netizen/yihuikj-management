# 后台管理系统-项目管理模块新增及修改需求（聚焦开发实现）

# 一、需求概述

本次需求针对后台管理系统「项目管理模块」进行功能新增及逻辑优化，核心涵盖四大模块：项目编辑权限控制、成本支出列表优化、新增资金管理区域、接口适配调整。

核心前提：1. 数据库表及字段由Gemini自行设计，需贴合现有系统数据库规范（腾讯云开发集合、Firebase蓝图），确保与现有系统（用户、客户、项目、成本、凭证等相关数据）无缝对接，数据可管理、无异常；2. 所有新增功能、交互、接口需与现有系统逻辑、布局、样式保持统一，不破坏原有系统功能；3. 仅新增必要功能及接口，不冗余开发。

# 二、具体功能需求（详细可直接落地，无需额外推导）

## 2.1 已结清状态项目编辑权限控制（核心开发点：权限拦截+只读控制）

适用范围：所有已创建成功的项目（含新建单子、补录单子，补录单子需与现有系统补录项目标识逻辑保持一致），且项目状态为「已结清」（需与现有系统项目状态体系匹配，对应系统配置中“已结清”状态）。

- 编辑权限明确：点击「编辑」按钮后，仅开放以下4项内容的编辑权限，无任何其他编辑入口，确保权限管控严谨：
        

    - 项目名称：可修改，修改后同步更新至项目相关关联展示场景，与现有项目名称编辑逻辑一致。

    - 项目描述：可修改，支持多行输入、文本格式化，与现有项目描述编辑逻辑一致。

    - 成本支出：支持新增、编辑、删除成本项，成本项的编辑规则（如类目选择、金额录入）与现有成本支出管理逻辑完全一致，不新增额外限制。

    - 凭证上传：支持新增、删除项目相关凭证（图片、单据等），上传规则（文件类型、大小限制、存储路径）与现有凭证上传逻辑一致，支持预览凭证。

- 禁止编辑范围明确：除上述4项外，其余所有项目相关信息均禁止编辑，字段统一设为只读状态，具体包括但不限于：
        

    - 项目基础信息：项目类型（新建/补录/长期）、关联客户、项目周期（开始/结束日期）、施工周期、回款周期等。

    - 项目状态相关：不可修改项目状态（已结清）及状态相关标识、展示文本。

    - 资金相关：后续新增的资金管理区域中，自动计算类字段不可编辑（已明确，详见2.3条款）。

- 权限校验要求：前端需隐藏禁止编辑字段的编辑入口，hover无编辑提示；后端接口需增加项目状态判断拦截，若项目为已结清状态，接收非法编辑参数（禁止编辑的字段）时，需返回清晰异常提示，双重校验确保权限不被突破，与现有系统权限校验风格一致。

## 2.2 成本支出管理列表优化（核心开发点：新增栏目+默认值+联动计算）

- 新增栏目：在「成本支出管理」列表中，新增「是否已结清」栏目，栏目位置需与现有栏目（如成本类目、供应商、金额等）布局协调，样式（字体、对齐方式、间距）与现有栏目完全统一，不突兀。

- 栏目交互：仅支持两个固定选项——「是」「否」，采用现有系统下拉框或单选按钮样式，无其他可选值，点击可切换选项，切换后实时触发后续资金管理区域“已付账款”的重新计算（联动逻辑详见2.3.4）。

- 默认值规则：新增成本支出项时，「是否已结清」栏目默认选中「是」，无需用户手动选择，用户可根据实际情况修改为「否」，默认值需在新增表单中预先选中，与现有表单默认值逻辑一致。

- 联动逻辑：该栏目取值直接决定“已付账款”的计算范围，仅筛选「是否已结清」为「是」的成本支出项，纳入已付账款计算（计算规则详见2.3.4），切换选项后需实时同步更新资金管理区域对应字段，无延迟。

- 历史数据兼容：现有已存在的成本支出项，需自动为「是否已结清」栏目赋值为「是」，确保历史数据不缺失，不影响现有成本支出展示及后续资金计算。

## 2.3 新增资金管理区域（核心开发点：区域布局+字段迁移+自动计算+实时同步）

- 区域基础配置：
        

    - 区域名称：固定命名为「资金管理」，命名风格与现有模块（基础项目信息、成本支出管理）统一，标题样式（字体、大小、颜色）与现有区域标题一致。

    - 区域位置：固定位于「基础项目信息」区域与「成本支出管理」区域之间，布局结构（边框、内边距、字段排列方式）与现有两个区域完全一致，确保视觉统一、排版流畅，无错位、重叠。

    - 适配要求：区域适配现有系统页面响应式布局，在不同屏幕尺寸下，字段排列、样式保持一致，不出现布局错乱。

- 字段配置及规则（含字段迁移）：
        

    - 字段1：订单金额（迁移字段）
                

        - 迁移要求：将原有「基础项目信息」区域中的「订单金额」字段，完整迁移至「资金管理」区域，字段样式、编辑权限、必填属性与原有完全一致，不改变原有录入规则（如必填、金额格式限制）。

        - 展示要求：字段标签明确标注「订单金额」，与其他资金相关字段排列整齐，录入框样式与现有金额录入框一致，支持金额格式化（保留2位小数）。
        - 格式要求：跟钱有关的输入框，只能输入数字、小数点、负号等有效数字，禁止输入其他字符。

    - 字段2：已收账款（新增可编辑字段）
                

        - 交互规则：用户可手动录入已收到的账款金额，支持小数录入（保留2位小数），禁止录入负数、非数字内容，录入时实时触发“未收账款”的重新计算。

        - 默认值：新增项目时，默认值为0，无需用户手动输入；编辑项目时，保留上次录入值，可修改。

        - 校验规则：录入金额不可超过订单金额，若超出则实时弹出校验提示（提示内容简洁明确，样式与现有系统校验提示一致），禁止提交，直至修正。

    - 字段3：未收账款（新增自动计算字段，禁止编辑）
                

        - 计算逻辑：未收账款 = 订单金额 - 已收账款，系统实时自动计算，无需用户手动输入，计算结果保留2位小数，非负。

        - 展示规则：字段标签标注「未收账款」，同时标注“自动计算，不可编辑”字样（标注样式与现有系统自动计算字段一致），字段设为只读状态，无编辑入口，hover无编辑提示。

        - 异常处理：若已收账款＞订单金额，计算结果为负数时，不展示负数，仅弹出校验提示（与已收账款录入校验提示一致），同时该字段暂不展示计算结果，直至录入金额修正。

    - 字段4：应付账款（新增自动计算字段，禁止编辑）
                

        - 计算逻辑：应付账款 = 当前项目下所有成本支出项的金额总和，系统实时自动计算，无需用户手动输入，计算结果保留2位小数，非负。

        - 展示规则：同未收账款，标注“自动计算，不可编辑”，只读状态，无编辑入口；成本支出项新增、编辑、删除时，实时同步更新计算结果，无延迟。

    - 字段5：已付账款（新增自动计算字段，禁止编辑）
                

        - 计算逻辑：已付账款 = 当前项目下所有「是否已结清」为「是」的成本支出项金额总和，系统实时自动计算，无需用户手动输入，计算结果保留2位小数，非负，且不超过应付账款金额。

        - 展示规则：同其他自动计算字段，标注“自动计算，不可编辑”，只读状态；「是否已结清」栏目切换、成本支出项新增/编辑/删除时，实时同步更新计算结果，无延迟。

        - 异常处理：若计算结果超出应付账款金额，弹出异常提示（样式与现有系统一致），同时暂不更新该字段，排查成本支出数据后自动修正。

- 数据同步要求：所有资金相关字段（订单金额、已收账款）修改后，相关自动计算字段（未收、应付、已付账款）需在1秒内完成更新，确保数据实时一致，无延迟、无计算错误。

## 2.4 接口适配调整（核心开发点：接口新增/调整+权限控制+数据同步，数据库字段由Gemini自行设计）

核心要求：接口路径、命名风格、请求/响应格式与现有系统完全一致，兼容现有前端调用逻辑，不新增冗余参数，确保接口调用无异常；数据库字段、表结构自行设计，接口需适配自行设计的数据结构，实现以下功能。

1. 项目创建接口（沿用现有接口路径，仅调整参数）
        

    - 新增参数：已收账款（对应资金管理区域的已收账款字段），可选填，默认值为0，录入规则与前端一致（非负、保留2位小数）。

    - 参数关联：订单金额参数仍为必填项，与已收账款参数联动校验（已收账款≤订单金额），校验失败返回异常提示。

    - 额外逻辑：接口调用成功后，自动同步创建资金相关关联数据，自动计算未收、应付、已付账款（应付、已付账款初始为0，无成本支出时），确保项目创建后，资金管理区域数据正常展示。

    - 原有逻辑不变：保留现有项目创建的所有参数、校验规则、返回值，不破坏原有创建逻辑。

2. 项目编辑接口（沿用现有接口路径，重点增加权限控制）
        

    - 权限拦截逻辑：新增项目状态判断，若项目状态为「已结清」，仅接收以下参数：项目名称、项目描述、成本支出相关参数（新增/编辑/删除成本项）、凭证上传相关参数，其他参数（如项目类型、项目周期、订单金额等）均拦截，返回异常提示（明确告知“已结清项目仅可编辑指定内容”）。

    - 资金相关参数：允许接收已收账款参数（可修改），禁止接收未收、应付、已付账款相关参数（自动计算字段，不可手动修改）；若修改订单金额，需同步校验已收账款≤订单金额，校验失败返回异常提示，同时自动重新计算未收账款。

    - 同步逻辑：接口调用成功后，同步更新资金相关数据，确保资金管理区域字段实时同步，无数据偏差。

3. 成本支出新增/编辑接口（沿用现有接口路径，仅新增参数）
        

    - 新增参数：是否已结清（可选值为“是”/“否”，默认值为“是”），参数取值需与前端栏目选项对应，禁止传入其他非法值，校验失败返回异常提示。

    - 同步逻辑：接口调用成功后，实时触发资金管理区域应付账款、已付账款的重新计算，并同步更新对应数据，确保成本支出与资金数据实时一致。

    - 原有逻辑不变：保留现有成本支出新增/编辑的所有参数、校验规则（如金额非负、类目匹配等），不破坏原有逻辑。

4. 成本支出删除接口（沿用现有接口路径，增加同步逻辑）
        

    - 新增逻辑：接口调用成功后，实时触发资金管理区域应付账款、已付账款的重新计算，同步更新对应数据，确保删除成本项后，资金数据同步修正。

5. 项目详情查询接口（沿用现有接口路径，新增返回参数）

    - 新增返回参数：资金管理相关所有字段（订单金额、已收账款、未收账款、应付账款、已付账款），返回格式与现有金额字段一致（保留2位小数），用于前端资金管理区域展示。

    - 补充返回参数：成本支出列表中，新增「是否已结清」字段返回值，与前端栏目选项对应（如“是”/“否”），便于前端展示。

    - 原有返回参数不变：保留现有项目详情的所有返回字段，确保前端现有展示逻辑不受影响。

6. 资金计算同步接口（新增接口，应对异常场景）
        

    - 接口功能：手动触发指定项目的资金相关字段（未收、应付、已付账款）重新计算，用于解决数据同步异常（如延迟、计算错误）场景。

    - 请求参数：projectId（必填，项目唯一标识），用于指定需要同步的项目。

    - 返回逻辑：同步计算完成后，返回同步结果（成功/失败），失败时返回具体错误原因，便于排查；同步过程中校验数据合法性（如未收账款非负、已付账款≤应付账款），异常时返回校验提示。

7. 历史数据同步接口（新增接口，兼容历史项目）
        

    - 接口功能：为现有所有历史项目（已创建成功，未包含资金管理数据的项目），自动补充资金相关数据，确保历史项目的资金管理区域正常展示、计算。

    - 请求参数：可选参数projectId（指定单个项目同步），不传入参数则同步所有历史项目。

    - 同步规则：同步时，订单金额沿用项目原有订单金额；已收账款默认设为0；未收账款自动计算（订单金额-0）；应付账款自动计算（该项目所有成本支出金额总和）；已付账款自动计算（该项目「是否已结清」为「是」的成本支出金额总和）；同步过程中备份历史数据，避免数据丢失。

    - 返回逻辑：返回同步结果（成功数量、失败数量、失败项目ID及原因），便于排查异常。

## 2.5 数据同步及异常处理（核心开发点：实时同步+异常拦截+日志记录）

- 同步场景全覆盖：以下场景需触发资金相关数据同步，确保数据一致：
        

    - 订单金额、已收账款修改时，同步更新未收账款。

    - 成本支出项新增、编辑、删除时，同步更新应付账款、已付账款。

    - 成本支出项「是否已结清」栏目切换时，同步更新已付账款。

    - 手动调用资金计算同步接口时，同步更新所有自动计算字段。

- 异常拦截及提示：所有可能出现的异常场景，需拦截并返回清晰提示，具体包括：


    - 已收账款＞订单金额：录入时实时提示“已收账款不可超过订单金额，请修正”。

    - 已付账款＞应付账款：计算异常时提示“已付账款不可超过应付账款，请检查成本支出数据”。

    - 非法参数录入：如非数字金额、「是否已结清」传入非法值，提示“参数录入错误，请修正”。

    - 数据同步失败：提示“数据同步异常，请手动触发同步或联系管理员”。

- 日志记录要求：所有资金相关操作（自动同步、手动同步、异常拦截）均需记录日志，日志需包含：操作时间、操作人、操作类型（新增/编辑/同步）、项目ID、操作前后数据、操作结果（成功/失败），便于后续问题排查，日志格式与现有系统日志一致。

# 三、交互与样式要求（完全贴合现有系统，无需新增风格）

- 只读字段样式：所有禁止编辑的字段（含已结清项目的禁止编辑字段、资金管理区域自动计算字段），样式与现有系统只读字段完全一致（如灰色显示、无编辑边框、hover无编辑提示），不新增额外样式。

- 资金管理区域布局：严格遵循现有区域布局规范，字段排列整齐，标签与录入框/展示框的间距、对齐方式，与基础项目信息、成本支出管理区域一致；自动计算字段的“不可编辑”标注，与现有系统标注样式一致。

- 成本支出列表：新增「是否已结清」栏目，样式（字体、颜色、间距、选项样式）与现有栏目统一，切换选项时的交互效果（如选中状态、hover效果）与现有列表交互一致。

- 校验提示：所有异常校验提示，样式（颜色、边框、位置）、提示语气与现有系统完全一致，不突兀，确保用户体验统一。

- 响应式适配：所有新增区域、栏目、字段，需适配现有系统响应式布局，在电脑端、平板端等不同屏幕尺寸下，布局流畅、无错位、无样式错乱。

# 四、开发注意事项（聚焦开发落地，无数据库设计相关内容）

1. 权限控制严谨性：已结清项目的编辑权限需双重校验（前端隐藏+后端拦截），杜绝通过前端漏洞、非法接口调用修改禁止编辑的字段，确保权限管控无遗漏。

2. 数据计算准确性：资金相关自动计算逻辑需严谨，确保所有场景下计算结果正确（保留2位小数、非负、联动同步），避免出现计算错误、数据延迟，新增异常拦截逻辑，覆盖所有可能的异常场景。

3. 现有系统兼容性：所有新增/调整的功能、接口，需与现有系统无缝对接，不破坏现有任何功能（如项目创建、成本支出管理、凭证上传等），确保现有系统正常运行，不出现线上故障。

4. 历史数据兼容：通过历史数据同步接口，确保所有历史项目均能正常展示资金管理数据，同步过程中备份数据，避免数据丢失、数据错乱，同步完成后需测试历史项目的资金计算、展示是否正常。

5. 接口兼容性：新增/调整的接口，需兼容现有前端调用逻辑，不改变现有接口的请求/响应格式，仅新增必要参数、返回值，确保前端无需额外修改即可适配（除资金管理区域展示相关前端调整外）。

6. 交互一致性：所有新增交互（如栏目切换、字段编辑、校验提示），需与现有系统交互风格完全一致，不新增额外交互方式，确保用户操作无学习成本。

7. 测试覆盖：开发完成后，需覆盖所有场景测试，包括：新增项目、编辑项目（已结清/未结清）、成本支出新增/编辑/删除、凭证上传/删除、资金计算同步、历史数据同步、异常场景模拟，确保功能稳定可用。

# 五、核心开发要点

- 权限控制：已结清项目仅可编辑「项目名称、项目描述、成本支出、凭证上传」，其余字段只读，双重校验。

- 成本列表：新增「是否已结清」栏目（是/否，默认是），切换联动已付账款计算。

- 资金管理区域：位于基础信息与成本支出之间，迁移订单金额，新增已收（可编辑）、未收/应付/已付（自动计算，不可编辑），实时同步计算。

- 接口：适配现有风格，新增/调整参数及逻辑，新增资金同步、历史数据同步接口，确保数据一致。

- 异常处理：拦截所有非法场景，记录操作日志，确保数据准确、系统稳定。

- 兼容性：贴合现有系统样式、逻辑，兼容历史数据，不破坏现有功能。
